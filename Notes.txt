****What Hadoop is??****
Big data = very large data (TBs, PBs)
Normal System store it in a single machine
But Hadoop store it in distributed way(Many machines)

For example:-
1 CPU -> Slow
3 CPUs -> Split work and can work fast

****What is HDFS(Important)****
HDFS -> Hadoop distributed file system
It stores data across multiple machines

Components:-
NameNode -> Brain (Metadata -> Data about data)
DataNode -> It stores actual data


****Practical Implementation****

(i)JPS
It checks whether hadoop is working or not
It prints:
NameNode
DataNode
SecondaryNameNode
ResorceManager
NodeManager

But before this, we have to start the hadoop services using:-
start-all.sh (It starts all of the services)

Manually you can type:-
start-dfs.sh -> Hadoop Services

(ii)List files in HDFS
hdfs dfs -ls /

(iii)Creating a directory in HDFS
hdfs dfs -mkdir /university

(iv)How to create multiple directories using single command
hdfs dfs -mkdir -p /university/students /university/faculty /university/data
-p -> It ensures that if in case university directory didn't exist than it will first create the university directory and than its sub-directories

Without -p → parent must already exist
With -p → Hadoop will create parent also


(v)Uploading file to HDFS(put command)
Firstly we need to create a file in our local system.
-touch info.txt
-echo "Name:Amandeep Singh, Course:BTech, Subject:Hadoop" > info.txt
-cat info.txt

-hdfs dfs -put info.txt /university/students

Using put command you can upload files from local system to HDFS

(vi)Read File from HDFS
-hdfs dfs -cat /university/students/info.txt

(vii)Uploading multiple files
-echo "Data file 1" > file1.txt
-echo "Data file 2" > file2.txt
-hdfs dfs -put file1.txt file2.txt /university/data

(viii)Copy File in HDFS
-hdfs dfs -cp /university/students/info.txt /university/faculty

cp = duplicate

(ix)Move File in HDFS (It moves file form one location to another, File is removed from the source)
**It is used for renaming a file also
(-hdfs dfs -mv /temp/temp.txt /temp/temp1.txt) -> File name changed to temp1.txt

-hdfs dfs -mv /university/students/info.txt /university/data

mv = cut + paste

(x)Check Disk Usage (du)
-hdfs dfs -du /university

It shows size of files inside directory
Output -> File size + Path

(xi)Download File from HDFS (get command)
-hdfs dfs -get /university/data/info.txt .

. -> Means current directory

(xii)Delete File (rm)
-hdfs dfs -rm /university/data/info.txt

Deletes only file, Directory remains

(xiii)Delete Directory
-Case 1: Empty Directory

hdfs dfs -rmdir /university/faculty

-Case 2: Non-Empty Directory

hdfs dfs -rm -r /university/data

Deletes everything inside + directory

(xiv)Head command(View first few lines)
-hdfs dfs -head /university/students/info.txt

(xv)Tail command(View last few lines)
-hdfs dfs -tail /university/students/info.txt

(xvi)Count files and directories
-hdfs dfs -count /university
DIR_COUNT, FILE_COUNT, CONTENT_SIZE PATH

(xvii)Stat command(Shows specific info about a path)
**It works with directories also (For directories, it shows only name and Modification time, rest everything zero)

-%n: File Name
-%b: File size (bytes)
-%o: Block size
-%r: Replication Factor (number of copies of a file block stored across DataNodes)
-%y: Modification Time

-hdfs dfs -stat %n /university/students/info.txt

(xviii)Group Permission(chmod)
-Change mode(permissions) of a file or a directory
-User:Group:Others (7:5:5)
-4:Read(r), 2:Write(w), 1:Execute

-hdfs dfs -chmod 755 /university/students/info.txt

(xix)Owner and Group Change(chown)
-hdfs dfs -chown amandeep:group1 /university/students/info.txt (It will change name and group)
-hdfs dfs -chown amandeepsingh /university/students/info.txt (It will change only name)
-hdfs dfs -chown :Gamer_Group /university/students/info.txt (It will change only group)

(xx)Only Group Change(chgrp)
-hdfs dfs -chgrp Study_Group /university/students/info.txt

It will change only Group

chown:- Full control over user and group
chgrp:- Control over only group

(xxi)File exist or not(-test -e)
-0: Exists
-1: Does not exist
-echo $?: It shows the exit status of the last executed command

hdfs dfs -test -e /university/students/info.txt
echo $?

(xxii)Directory exist or not(-test -d)
-hdfs dfs -test -d /university
-hdfs dfs -test -e /university
Both will return 0(Exists)

They will differ, if the path is not a directory
-hdfs dfs -test -d /university/students/info.txt (Path is file -> It will return 1)
-hdfs dfs -test -e /university/students/info.txt (Return false)

(xxiii)Verify File integrity(checksum)
-hdfs dfs -checksum /university/students/info.txt

Gives hash/checksum

(xxiv)HDFS storage space (df command)
It tells about the storage space in HDFS
-df ->	total filesystem space
-du ->	size of specific file/folder

-hdfs dfs -df /
-hdfs dfs -df -h / (Gives in Human Readable format)

(xxv)Append data to HDFS file (Local file to HDFS file)
-appendToFile

-hdfs dfs -appendToFile localFile.txt /temp/temp.txt (Data will be appended at the EOF)

****Understanding Java Code****
Hadoop always works in 3 parts:-
(i)Mapper
Takes input → processes it → gives output (key, value)
(ii)Reducer
Takes mapper output → combines it → final result
(iii)Driver(Main)
Connects everything (Mapper + Reducer + Input + Output)


Firstly lets understand about the Mapper Class

public static class TokenizerMapper
    extends Mapper<Object, Text, Text, IntWritable> // <Input Key, Input Value, Output Key, Output Value>

In this we tell what type of data will come IN and what type of data will go Out


InputKey -> It represents the line number or byte offset(Use OBJECT in InputKey everytime)
InputValue -> Actual data from the file (Text)

OutputKey -> If output key will be in NUMERIC FORM use IntWritable else Text
OutputValue -> Same for this

Reducer Class

public static class Reducer
    extends Reducer<Text, IntWritable, Text, IntWritable> // <Input Key, Input Value, Output Key, Output Value>

Last two parameters of the Mapper class will be same for the first two parameters of the Reducer class
